復習：図で思い出す（参照と更新の分岐）

executeQuery と executeUpdate

第4回の更新系（INSERT / UPDATE / DELETE）は executeUpdate と 更新件数（int）です。参照の executeQuery・ResultSet とは役割が違います。

トランザクションの型（お約束）

  • 複数の SQL を1本の Connectionで順に実行する（メソッドごとに別接続だとトランザクションにまとめられない）。
  • setAutoCommit(false) で自動コミットを止める。
  • 成功なら commit()、失敗なら rollback()。
  • finally で setAutoCommit(true) に戻す。

第3回とのつながり（SQL の安全な渡し方）

更新系も文字列連結ではなく ? と setXxx で値を渡す、という原則は第3回と同じです。

復習チェック（口頭で確認）

  • 参照と更新で、executeQuery / executeUpdate と戻り値の違いを言える。
  • commit と rollback を、いつ呼ぶか一言で説明できる。
  • なぜ「複数 SQL を同じ Connection で」まとめる必要があるか、トランザクションの観点で言える。
  • EmpDao の各メソッドが内部で getConnection() している場合、発展の「1トランザクション複数操作」ではどう工夫するか、本編の言葉で思い出せる。
