DAO 練習問題（更新系・トランザクション）〜INSERT / UPDATE / DELETE〜

進め方（3ステップ）

  1. Step 1：講義テキスト（01-テキスト-DAO更新とトランザクション.html）の穴埋めと EmpDao の更新系メソッドの流れを確認する。
  2. Step 2（基本）：EmpDao の insert / delete / updateSalaryAndDept を 別々の実行で試す（各呼び出しが独自に接続を取る）。
  3. Step 3（発展）：同一 Connection で、指定順の6操作を 1トランザクションにまとめる。失敗時は rollback すること。

注意（emp_id の取り合い）

演習で新規に挿入する emp_id は、既存の 1〜10 や他の受講者と被らない番号を使うこと（例: 自分の出席番号、11〜99 の空き）。基本問題の解答例では 88 を仮に使う。終了前に テスト行を削除してよい状態に戻すこと。

基本問題（INSERT / DELETE / UPDATE）

問題1（INSERT）：EmpDao#insert で、新しい社員（emp_id は自分で決める。苗字・名前・給与・dept_no は任意）を1件追加し、Workbench または findById で確認せよ。

ヒント

new EmpDto(88, "演習", "太郎", 200000, 10, null) のように第6引数は null でよい。

解答例 
EmpDao dao = new EmpDao();
EmpDto e = new EmpDto(88, "演習", "太郎", 200000, 10, null);
int n = dao.insert(e);
System.out.println("insert件数=" + n);
System.out.println(dao.findById(88));

問題2（DELETE）：問題1で入れた emp_id の行を EmpDao#delete で削除し、削除件数と findById が null になることを確認せよ。

解答例 
EmpDao dao = new EmpDao();
System.out.println("delete件数=" + dao.delete(88));
System.out.println(dao.findById(88));

問題3（UPDATE）：既存の emp_id = 3 の給与を 290000、部署を dept_no = 30 に変更したあと、元の値（給与 280000、部署 20）に戻せ。

ヒント

updateSalaryAndDept は主キー emp_id を DTO に含める。

解答例 
EmpDao dao = new EmpDao();
dao.updateSalaryAndDept(new EmpDto(3, null, null, 290000, 30, null));
System.out.println(dao.findById(3).getSalary() + " " + dao.findById(3).getDeptNo());
dao.updateSalaryAndDept(new EmpDto(3, null, null, 280000, 20, null));

fname/lname は UPDATE 文に含まれないため null のままでよい。

発展問題（1トランザクション・6操作）

仕様：setAutoCommit(false) した 同一 Connection 上で、次をこの順に実行し、最後に commit する。途中で例外が出たら rollback し、finally で setAutoCommit(true) に戻す。

  1. INSERT（テスト用の社員。主キーは他と被らない値）
  2. SELECT（挿入した行が取れることを確認・表示）
  3. UPDATE（給与など1列でもよい）
  4. SELECT（更新後の値が見えることを確認）
  5. DELETE（テスト行を削除）
  6. SELECT（該当なしになることを確認）
ヒント

EmpDao の各メソッドは接続を都度取るため、この課題では Connection を1つ開いたまま prepareStatement を繰り返す実装にする。参考実装: Unit04_DAO_Update/src/TransactionSample.java（emp_id = 99）。

解答の参照先

リポジトリの Unit04_DAO_Update/src/TransactionSample.java をそのまま読み、自分の emp_id に書き換えて実行してよい。
