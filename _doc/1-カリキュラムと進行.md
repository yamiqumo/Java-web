## Java Webアプリケーション開発（フルバージョン）カリキュラム（Lesson20〜）

### 0. 前提とねらい

- **対象**:  
  - Java基礎〜オブジェクト指向（自販機CLI）を一通り学んだ受講者  
  - **理解の早いリーダー（1.0）**＋**初心者2名（0.5 × 2）** を想定した少人数チーム
- **前提**:
  - ここからは **完全に新しい「Webアプリ専用プロジェクト」** として開始する。
  - 既存の自販機CLIのロジック・クラス設計は **一切引き継がない**。
  - Webアプリ用に、**フォルダ構成・命名・I/Oの流れをゼロから設計し直す**。
- **ゴール**:
  - **HTMLフォーム → Servlet → JDBC/DAO → MySQL → JSP表示** の一連の「データの通り道（I/O）」を、**型（テンプレート）として暗記レベル**で身につける。
  - リーダーが自学したうえで、初心者向けに **「穴埋め問題（型）」を自作できる状態** にする。

---

### 1. 全体像：データの通り道（I/O）を最優先する

- **最重要テーマ**: 「データがどこから来て、どこを通り、どこに表示されるか」を常に意識する。
- **基本フロー（黄金ルート）**:
  1. **画面（HTMLフォーム）**: 入力フォームで値を受け取る（input, select, textarea）
  2. **HTTPリクエスト**: ブラウザからサーバへ（URL, method, パラメータ）
  3. **Servlet（Controller）**: `getParameter` で値を取り出し、Javaの型に変換
  4. **Model（JavaBean/DAO）**: 入力値を使ってDBアクセス（JDBC + PreparedStatement）
  5. **MySQL（DB）**: CRUD（SELECT/INSERT/UPDATE/DELETE）
  6. **Servlet**: 結果を `setAttribute` でJSPに渡す
  7. **JSP（View）**: `<%= %>` で値を埋め込み、`<table>` で一覧表示

これを **Lesson20〜26 すべてで貫く軸** とし、**I/Oから外れる話（細かいデザイン・フレームワーク特殊機能 等）は後回し**にする。

### 3. Lesson 20: HTML基礎（I/O特化）

#### 学習目標

- **入力と出力の枠**（フォーム＋テーブル）を、**Java無しで** 用意できる。
- `<form>` 要素と主な入力部品（`input`, `select`, `textarea`）の **基本パターン** を説明できる。
- `<table>` で **レコード一覧の形**（ヘッダ行＋データ行）を作成できる。
- 「この画面の入力値が、将来Servletに飛んでDBに入る」までの **イメージ** を、図で説明できる。

#### 覚えるべき型（定型コード）

- **フォームのひな型**（POST送信・名前とIDを統一）

```html
<form action="/app/sample" method="post">
  <label>商品名: <input type="text" name="itemName"></label>
  <label>価格: <input type="number" name="price"></label>
  <button type="submit">登録</button>
</form>
```

- **一覧表示のテーブル型**（ヘッダ固定＋データ行の繰り返し前提）

```html
<table border="1">
  <tr>
    <th>ID</th>
    <th>商品名</th>
    <th>価格</th>
  </tr>
  <!-- ここにデータ行が複数並ぶ想定 -->
  <tr>
    <td>1</td>
    <td>サンプル商品</td>
    <td>100</td>
  </tr>
</table>
```

- **画面レイアウトの考え方の型**:
  - **上部**: タイトル・説明
  - **中央左**: 入力フォーム
  - **中央右**: 結果テーブル（のちにJSPで動的化）
  - **下部**: エラー表示やメッセージの場所

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（フォーム型）**:  
  - 「商品登録フォーム」「会員登録フォーム」など **題材だけ変えたフォーム** を3パターン作る。  
  - 初心者には `action`, `method`, `name` 属性の一部を **`_____` にして穴埋め** させる。
- **ミッション2（テーブル型）**:  
  - 3種類の一覧（商品一覧・会員一覧・注文履歴一覧）テーブルのテンプレートを作る。  
  - `<th>` のラベルは完成済み、`<td>` の一部だけを穴埋めにする（列の意味を考えさせる）。
- **ミッション3（I/Oマッピング図）**:  
  - フォームの `name` とテーブルの列名を線で結ぶ **マッピング図の台紙** を用意し、空欄を埋める問題を作る。

### 4. Lesson 21: Webの仕組み & Tomcat導入

#### 学習目標

- **リクエスト/レスポンス** とは何かを、**図と一言説明**で話せる。
- URL, HTTPメソッド（GET/POST）、パラメータの関係を説明できる。
- Tomcatの **ディレクトリ構造とデプロイの型**（どこに何を置くか）を理解する。
- 簡単なServletをTomcat上で動かし、ブラウザからアクセスして確認できる。

#### 覚えるべき型（定型コード）

- **リクエスト〜レスポンスの説明テンプレート**
  - 「ブラウザが **リクエスト（お願い）** を送り、サーバが **レスポンス（お返事）** を返す」  
  - リーダーはこの「一文＋矢印図」を毎回使い回す。

- **Tomcat配下の基本構成（例）**

```text
webapps/
  myapp/               ← プロジェクトのルート
    WEB-INF/
      web.xml          ← ルーティング設定（入門では最小限）
    index.jsp          ← トップページ
    form.html          ← フォーム画面
```

- **最小Servletの型**

```java
@WebServlet("/hello")
public class HelloServlet extends HttpServlet {

	@Override
	protected void doGet(HttpServletRequest request, HttpServletResponse response)
			throws ServletException, IOException {

		response.setContentType("text/plain; charset=UTF-8");
		PrintWriter out = response.getWriter();
		out.println("Hello Servlet");
	}
}
```

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（図の穴埋め）**:  
  - 「ブラウザ → (_____) → サーバ → (_____) → ブラウザ」の図を作り、  
    `リクエスト` / `レスポンス` / `HTML` などの語を穴埋めにする。
- **ミッション2（URLとパスの対応）**:  
  - `http://localhost:8080/myapp/hello` がどのServletに届くかを説明する **対応表テンプレート** を作り、  
    `コンテキストパス` / `パス（/hello）` などを穴埋め。
- **ミッション3（Servletクラス穴埋め）**:  
  - 上記 `HelloServlet` から、`@WebServlet`, `doGet`, `PrintWriter`, `"Hello Servlet"` などを抜いた **穴埋め版** を作る。

### 5. Lesson 22: MySQL 基礎 (CRUD)

#### 学習目標

- RDBの基本用語（データベース / テーブル / レコード / カラム）を説明できる。
- **1テーブルに対するSELECT / INSERT / UPDATE / DELETE** を手で書いて実行できる。
- 主キー（id）を使った1件取得・更新・削除の流れを理解する。
- データの変更が **一覧表示にどう影響するか** を意識できる。

#### 覚えるべき型（定型コード）

- **テーブル定義の型（例：itemsテーブル）**

```sql
CREATE TABLE items (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  price INT NOT NULL
);
```

- **基本CRUD文の型**

```sql
-- 全件取得
SELECT id, name, price FROM items;

-- 新規登録
INSERT INTO items (name, price) VALUES ('サンプル商品', 100);

-- 更新（id指定）
UPDATE items SET name = '変更後', price = 200 WHERE id = 1;

-- 削除（id指定）
DELETE FROM items WHERE id = 1;
```

- **WHERE句の型**:  
  - `WHERE id = ?`  
  - `WHERE name LIKE ?`

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（CRUD穴埋め）**:  
  - 上記4種類のSQLから、テーブル名・カラム名・WHERE句を空欄にした問題を作る。  
  - 題材（items, members, orders など）だけ変えた問題を複数用意。
- **ミッション2（結果イメージ対応）**:  
  - 実行前後のテーブル状態を表にして、「どのSQLを打つとこうなるか」を選ばせる問題の **テンプレ台紙** を作る。
- **ミッション3（I/O視点の接続）**:  
  - 「フォームの入力値が、最終的にどのSQL文のどの部分に入るか」を線で結ぶ図の穴埋め問題を作る。

### 6. Lesson 23: JDBCの型 (クエリパラメータあり)

#### 学習目標

- JDBCを使った **DB接続〜SQL実行〜ResultSet処理** の一連の流れを理解する。
- `PreparedStatement` を使って **プレースホルダ（?）に値をバインド** できる。
- 接続情報（URL / ユーザ / パスワード）を **1か所にまとめるDAO/ユーティリティの型** を理解する。
- 例外処理・リソースクローズの基本パターンを写経レベルで再現できる。

#### 覚えるべき型（定型コード）

- **接続〜SELECTの最小パターン**

```java
String sql = "SELECT id, name, price FROM items WHERE id = ?";

try (Connection con = DriverManager.getConnection(URL, USER, PASS);
		PreparedStatement ps = con.prepareStatement(sql)) {

	ps.setInt(1, id); // パラメータをバインド

	try (ResultSet rs = ps.executeQuery()) {
		if (rs.next()) {
			int itemId = rs.getInt("id");
			String name = rs.getString("name");
			int price = rs.getInt("price");
			// ここでJavaのオブジェクトに詰める想定
		}
	}
} catch (SQLException e) {
	e.printStackTrace();
}
```

- **DAOメソッドの雛形**

```java
public Item findById(int id) {
	// 接続とSQLの型は上記と同じ
}
```

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（PreparedStatement穴埋め）**:  
  - `setInt(1, id)`, `executeQuery()`, `getString("name")` などを抜いた穴埋め版を用意。
- **ミッション2（カラム名と型の対応）**:  
  - `getInt`, `getString` のどちらを使うかを選ばせる対応表問題を作る。
- **ミッション3（DAOメソッド台紙）**:  
  - `findAll()`, `insert(Item item)`, `update(Item item)` のメソッドシグネチャだけ用意し、  
    中身を穴埋め（SQL文字列とパラメータ設定）にするテンプレートを作る。

### 7. Lesson 24: Servletの型（データの通り道）

#### 学習目標

- フォームから送られた値を `request.getParameter` で受け取り、**型変換（String→int等）** できる。
- 受け取った値をJavaオブジェクトに詰めて、DAOへ渡す流れを理解する。
- 処理結果（メッセージや一覧データ）を `request.setAttribute` でJSPに渡す黄金フローを説明できる。
- **1つのServletで「受け取り→処理→渡す」** までを一気通貫で書ける。

#### 覚えるべき型（定型コード）

- **フォーム入力の受け取り〜型変換〜Bean生成**

```java
String name = request.getParameter("itemName");
String priceStr = request.getParameter("price");
int price = Integer.parseInt(priceStr);

Item item = new Item();
item.setName(name);
item.setPrice(price);
```

- **処理結果をJSPへ受け渡す型**

```java
List<Item> itemList = service.findAll();
request.setAttribute("items", itemList);

RequestDispatcher dispatcher = request.getRequestDispatcher("/WEB-INF/jsp/itemList.jsp");
dispatcher.forward(request, response);
```

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（getParameter→型変換）**:  
  - `getParameter("____")` の引数や `Integer.parseInt(____)` の中身を穴埋めにした問題を作る。
- **ミッション2（属性名の統一）**:  
  - `setAttribute("items", itemList)` の `"items"` と、JSP側で使う変数名を揃える問題を用意。  
  - 「どの名前を統一すべきか」を考えさせる。
- **ミッション3（リクエストの通り道マップ）**:  
  - 「フォームのname → getParameterの引数 → Beanのフィールド名 → setAttributeのキー → JSPの `<%= %>`」を線で結ぶマップ台紙を用意し、穴埋めにする。

### 8. Lesson 25: JSPの型（表示のパズル）

#### 学習目標

- JSPで `<%= %>` を使って、Javaの値をHTMLに埋め込める。
- `<table>` の中で、Javaのリスト（List）を1行ずつ展開するパターンを理解する。
- エラーメッセージや確認メッセージを画面の決まった位置に表示できる。
- JSTLなどのタグライブラリに頼らず、**構造が見える書き方** を優先して理解する。

#### 覚えるべき型（定型コード）

- **単純な値の埋め込み型**

```jsp
<p>登録完了メッセージ: <%= message %></p>
```

- **一覧表示の基本型**

```jsp
<table border="1">
  <tr>
    <th>ID</th>
    <th>商品名</th>
    <th>価格</th>
  </tr>
  <% for (Item item : items) { %>
    <tr>
      <td><%= item.getId() %></td>
      <td><%= item.getName() %></td>
      <td><%= item.getPrice() %></td>
    </tr>
  <% } %>
</table>
```

- **メッセージ表示の位置決め型**

```jsp
<% if (message != null) { %>
  <p style="color: green;"><%= message %></p>
<% } %>
```

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（getter呼び出し穴埋め）**:  
  - `<%= item.get____() %>` の `Id/Name/Price` 部分を穴埋めにした問題を作る。
- **ミッション2（for文型）**:  
  - `for (Item item : items)` の `Item` / `item` / `items` のどれをどこに書くかを選ばせる穴埋めテンプレートを用意。
- **ミッション3（レイアウトのパズル）**:  
  - 完成画面のスクリーンショット風イメージと、不完全なJSPコードを渡し、  
    「どの `<%= %>` をどこに入れると同じ見た目になるか」を考えさせるパズル問題を設計する。

### 9. Lesson 26: MVCモデルの統合（整理整頓）

#### 学習目標

- MVCの役割分担（Model / View / Controller）を、**自分のプロジェクト構成に当てはめて説明**できる。
- JavaBean・DAO・Servlet・JSP をそれぞれどこに置くか、**フォルダ構成の型**を決められる。
- 「1機能＝1画面＋1Servlet＋1DAOメソッド」が基本、という設計方針を共有できる。
- 小さなWebアプリ（例：商品一覧＋登録）を **ゼロからフォルダごと組み立てる** ことができる。

#### 覚えるべき型（定型コード・構成）

- **フォルダ構成の型（例）**

```text
myapp/
  src/
    main/
      java/
        webapp/controller/   ← Servlet
        webapp/model/        ← JavaBean・DAO
  webapp/
    WEB-INF/
      jsp/                   ← JSPファイル
    assets/                  ← CSS, 画像 等
```

- **1機能の紐づけ型**
  - 画面: `itemList.jsp`
  - Servlet: `ItemListServlet`（URLパターン `/items`）
  - DAO: `ItemDao#findAll()`

#### 初心者向けの穴埋め問題の作成ミッション

- **ミッション1（ファイル配置パズル）**:  
  - いくつかのクラス/JSP名をカードにしておき、「どのフォルダに置くか」を選ばせるパズル問題を作る。
- **ミッション2（MVC対応表の穴埋め）**:  
  - URL, Servletクラス名, JSPファイル名, DAOメソッド名の **対応表テンプレート** を用意し、一部を空欄に。
- **ミッション3（ミニアプリ設計書）**:  
  - 「商品登録＋一覧」のような1画面アプリについて、  
    `画面名 / URL / Servlet / DAOメソッド / 使用テーブル` を書き込む設計書テンプレ台紙を作り、  
    初心者に穴埋めさせる。

### 10. リーダー（1.0）の運用ガイドライン

- **毎Lessonでの役割**
  - **(1) 自学**: まずリーダーが「完成コード」を1回、自分で通して完成させる。
  - **(2) 型の抽出**: 完成コードから「最低限覚えるべき型（定型コード）」だけを抜き出し、シンプルな形に削る。
  - **(3) 穴埋め化**: その型からキーワード・識別子・順番だけを隠して「初心者向けの穴埋め問題」に変換する。
- **I/Oファーストの徹底**
  - どのLessonでも、**先に「データの通り道」を確認 → 次に細かい言語仕様** の順で説明する。
  - 「この値はどこからきて、どこに入り、どこに出ていくのか？」を、  
    口頭＋ホワイトボード（あるいは図）で毎回確認する。
- **CLIとの切り離し**
  - 過去の自販機CLIのロジックは **一切持ち込まない**。  
  - あくまで「Webアプリ用の新しいI/Oの世界」として、**HTTPとHTMLを起点**に話を組み立てる。
