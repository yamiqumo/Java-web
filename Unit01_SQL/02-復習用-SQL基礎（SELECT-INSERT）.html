ブロック4：INSERT / UPDATE / DELETE と手順の型

読む（2〜3分）

データを変える3文の役割を整理します。

  • INSERT INTO ... VALUES：行を1件追加する。主キー（item_id）の重複に注意。
  • UPDATE ... SET ... WHERE：指定した行の値を変える。WHERE なし = 全行変更になるため必ず id を指定。
  • DELETE FROM ... WHERE：指定した行を削除する。WHERE なし = 全削除になるため必ず id を指定。

動画で確認すること：テーブルの「カラム（列）」と「行」の関係、INSERT でどの列に何を入れるかの指定。

問題8：item_id が 101 の unit_price を 110 に変更し、その後 SELECT で変わったことを確認するSQLを書いてください。

ヒント

UPDATE テーブル名 SET 列 = 新しい値 WHERE item_id = ○○; の形で書きます。WHERE なしは全行変更になるので必ず id を指定してください。

解答例 
-- UPDATE（1行だけ変更：WHERE で狙う）
UPDATE items
SET unit_price = 110
WHERE item_id = 101;

-- SELECT で確認（変わったか？）
SELECT item_id, category, item_name, unit_price
FROM items
WHERE item_id = 101;

問題9：item_id が 202 の行を削除し、その後 SELECT で行数が減ったことを確認するSQLを書いてください。

ヒント

DELETE FROM テーブル名 WHERE item_id = ○○; の形で書きます。WHERE なしは全削除になるので必ず id を指定してください。

解答例
