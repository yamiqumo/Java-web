SELECT・INSERT・DELETE・UPDATEについてそれぞれを説明できる

それぞれのSQLが「どんなときに、どんな働きをするか」を日本語で説明できる。

  • SELECT: 必要な情報を抽出して任意のカラムを表示させる。
  • INSERT: 任意の情報を登録できる。
  • DELETE: 任意の情報を削除できる。
  • UPDATE: 任意の情報を更新できる。

3-1. 用語の確認（最小）

  • カラム（列）：項目。例）item_name、unit_price
  • ロウ（行）：1件分のデータ（レコード）。
  • 主キー（Primary Key）：行を一意に特定するための値。例）item_id

SQL

正規化（概要）：現実のまとまりある情報を、意味ごとに分割してテーブルに整理します。

SQLの基本文（SELECT / WHERE / ORDER BY）

SELECT：どのカラムを「見る」か決める

SELECT は「どのカラムを、どのテーブルから取り出して表示するか」を決める文です。SELECT * は「全部のカラム」、カンマ区切りで列名を書くと「必要なカラムだけ」取り出せます。

-- すべてのカラムを表示
SELECT * FROM items;

-- 一部のカラムだけ表示
SELECT item_id, item_name
FROM items;
  • * は最初の練習では便利だが、本番では必要なカラムだけ列挙するのが基本。
  • カラム名の順番は、画面に出てくる列の順番にもなる。

WHERE：条件でしぼり込む

WHERE は「どの行を対象にするか」を決める条件の文です。WHERE 列 演算子 値 の形で書きます。

SELECT *
FROM items
WHERE category = 'drink';

主な演算子（数値・文字に共通）

  • =：等しい（例：unit_price = 100）
  • < / <=：より小さい / 以下
  • > / >=：より大きい / 以上
  • <>：等しくない（!= を使うDBもある）

BETWEEN：範囲でしぼり込む

-- 100円以上 150円以下
SELECT *
FROM items
WHERE unit_price BETWEEN 100 AND 150;
  • BETWEEN は「両端を含む」範囲（以上かつ以下）になる。
  • 日付でも同じ書き方ができる（例：WHERE order_date BETWEEN '2025-04-01' AND '2025-04-30'）。

LIKE とあいまい検索

-- 「茶」で終わる飲み物
SELECT *
FROM items
WHERE item_name LIKE '%茶';
  • %：0文字以上の「なんでも」（前方一致・後方一致・部分一致に使う）。
  • _：1文字だけ「なんでも」。

IS NULL：NULL（まだ値が入っていない）の判定

SELECT *
FROM items
WHERE some_column IS NULL;

SELECT *
FROM items
WHERE some_column IS NOT NULL;
  • NULL は「0」でも「空文字」でもなく、「まだ値が入っていない・わからない」という特別な値。
  • NULL との比較は = ではなく IS を使う（= NULL と書くとヒットしない）。

AND / OR：条件を組み合わせる

SELECT *
FROM items
WHERE category = 'drink'
AND unit_price > 100;
  • AND：両方の条件を満たす行だけ残す。
  • OR：どちらか一方でも満たす行を残す（条件がゆるくなりやすいので注意）。

ORDER BY：並び順を決める

ORDER BY は「結果をどの順番で並べるか」を決める文です。書く位置は WHERE のあと になります。

-- 単価の高い順（降順）
SELECT *
FROM items
ORDER BY unit_price DESC;

-- スナックだけを、安い順（昇順）に並べる
SELECT *
FROM items
WHERE category = 'snack'
ORDER BY unit_price ASC;
  • ASC：昇順（小さい→大きい）。省略するとASCになる。
  • DESC：降順（大きい→小さい）。
  • 複数カラムで並べたいときは、カンマでつなぐ（例：ORDER BY category ASC, unit_price DESC）。

次は、データを変える3文

  • INSERT：行を追加。例：INSERT INTO items ( ... ) VALUES ( ... );（item_id が重複すると失敗しやすい）
  • UPDATE：行を更新。基本は UPDATE items SET ... WHERE item_id = ...;（WHEREなしは全行更新になりやすい）
  • DELETE：行を削除。基本は DELETE FROM items WHERE item_id = ...;（WHEREなしは全削除になりやすい）
-- INSERT（行を追加）
INSERT INTO items (item_id, category, item_name, unit_price) 
VALUES (303, 'drink', 'サンプル2', 140);

-- UPDATE（1行だけ変更：WHEREで狙う）
UPDATE items
SET unit_price = 110
WHERE item_id = 101;

-- DELETE（1行だけ削除：WHEREで狙う）
DELETE FROM items
WHERE item_id = 202;

3-4. INSERT / SELECT / UPDATE / DELETE の型（最小）

この回は、次の並びを「同じ順番で」たどれることがゴールです。

INSERT ▶ SELECT ▶ UPDATE ▶ SELECT ▶ DELETE ▶ SELECT
-- INSERT（行を追加）
INSERT INTO items (item_id, category, item_name, unit_price)
VALUES (999, 'drink', 'サンプル', 100);

-- SELECT（見る：条件/並び/行数）
SELECT item_id, category, item_name, unit_price
FROM items
WHERE category = 'drink'
ORDER BY item_id
LIMIT 10;

-- UPDATE（1行だけ変更：WHEREで狙う）
UPDATE items
SET unit_price = 110
WHERE item_id = 101;

-- DELETE（1行だけ削除：WHEREで狙う）
DELETE FROM items
WHERE item_id = 202;

SQLの書き方には、流れとしての「順番の型」があります。下のテンプレートは、その流れを整理したものです。

【手順テンプレ】
1) 初期データを入れる（INSERT）
2) 一覧を確認する（SELECT）
3) 1行だけ変更する（UPDATE）
4) もう一度確認する（SELECT）
5) 1行だけ消す（DELETE）
6) 最後に確認する（SELECT）

【メモテンプレ（結果を書き写す）】

変更した内容： unit_price を ___ → ___ にした
削除した行： item_id = ___
最終の行数：___ 行（増えた？減った？）

初期データ投入SQL（コピペ用）

この枠のSQLをコピーして、手元のツールに貼り付けて利用できます。

使うSQL（写経してから実行） 

-- 1) 見る（まず確認）
SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;

-- 2) 変更する（単価を1つだけ変える）
UPDATE items
SET unit_price = 110
WHERE item_id = 101;

-- 3) もう一度見る（変わった？）
SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;

-- 4) 消す（1行だけ消す）
DELETE FROM items
WHERE item_id = 202;

-- 5) 最後に見る（減った？）
SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;
ヒント
  • 最初は SELECT * で「見える化」してから始める。
  • UPDATE / DELETE は WHERE（id指定） を必ず付ける。
解答例

【解説・確認用】

正解は「変わった／消えた」を確認できることです。WHEREなしのUPDATE/DELETEは危険なので、必ずid指定を付けます。

SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;
UPDATE items SET unit_price = 110 WHERE item_id = 101;
SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;
DELETE FROM items WHERE item_id = 202;
SELECT item_id, category, item_name, unit_price FROM items ORDER BY item_id;

5. まとめ（写経で締める） 

SELECT = 取り出して見る（結果は表示用の表）
INSERT = 1行追加（次のSELECTで増えた分が見える）
UPDATE = 1行変更（WHEREで狙って変える）
DELETE = 1行削除（WHEREで狙って消す）
DB = 最後に残る箱（消えない）
