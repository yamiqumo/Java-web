環境構築：MySQL のセットアップとDB接続手順

0. 今回の環境について

この研修では、MySQL を利用してデータベースの操作を学びます。

  • DBMS（データベース管理システム）として MySQL を使用します。
  • 接続時のパスワードは password としています。
    ※ 本番環境では絶対に使わない弱いパスワードです。今回は研修環境のため、この設定で構築しています。
  • 操作はすべて コマンドプロンプト から行います。

1. MySQL のインストール

手順1：インストーラのダウンロード

  1. 以下のURLをブラウザで開く。
    https://downloads.mysql.com/archives/installer/
  2. Product Version を 8.0.43、Operating System を Microsoft Windows に合わせる。
  3. Windows (x86, 32-bit), MSI Installer（354.3M のフルサイズ版: mysql-installer-community-8.0.43.0.msi）の「Download」をクリックする。
    ※ 上の 2.1M は Web インストーラ（後でダウンロードが走る）です。354.3M のフルサイズ版を選んでください。

手順2：インストーラの実行

  1. ダウンロードした mysql-installer-community-8.0.43.0.msi をダブルクリックして起動する。
  2. 「Choosing a Setup Type」画面で Full を選択する。
    （MySQL Server・Workbench・JDBC ドライバなどが一括でインストールされます）
  3. 以降は「Next」または「Execute」をクリックして進める（次の手順3で注意箇所があります）。

手順3：Authentication Method の設定（重要）

ここだけ注意！ デフォルトのまま「Next」を押すと JDBC 接続でエラーになります。必ず下記の手順で設定を変更してください。

  1. 「Authentication Method」画面が表示されたら、Use Legacy Authentication Method (Retain MySQL 5.x Compatibility) を選択する。
  2. 「Next」をクリックして進む。
  3. 「Accounts and Roles」画面で root パスワード に password を入力する。
    ※ 研修環境用の設定です。本番では使わないパスワードです。
  4. その後は「Next」→「Execute」→「Finish」で完了。

手順4：Windows PATH への登録

インストール後、コマンドプロンプトから mysql コマンドを実行できるよう、実行ファイルのパスを Windows に登録します。

  1. スタートボタンを右クリックして「検索」を開き、システム環境変数の編集 と入力して開く。
  2. 「システムのプロパティ」が開いたら 「環境変数」 をクリック。
  3. 「システム環境変数」欄の 「Path」 を選択して 「編集」 をクリック。
  4. 「新規」をクリックし、以下のパスを入力する。
C:\Program Files\MySQL\MySQL Server 8.0\bin
  1. 「OK」→「OK」→「OK」で閉じる。
  2. 新しいコマンドプロンプトを開いて（既に開いているものは閉じてから）、以下を実行して確認する。
mysql --version

以下のように表示されれば登録成功です。

mysql  Ver 8.0.43 Distrib 8.0.43, for Win64 (x86_64)

2. DB の立ち上げ方法（コマンドプロンプト）

コマンドプロンプトを開き、次のコマンドで MySQL に接続します。

mysql -u root -p
  • -u root：ユーザー名を root で接続する指定。
  • -p：パスワードを入力するよう求めるオプション。実行後にパスワード（password）を入力する。
  • 接続が成功すると mysql> というプロンプトが表示される。

3. 現状の確認コマンド（DB一覧・テーブル一覧）

MySQL に入ったあと、まず「今どんな DB やテーブルがあるか」を確認する3つのコマンドを覚えましょう。

-- 存在するDB（データベース）の一覧を表示
SHOW DATABASES;

-- 利用するDBを指定する
USE データベース名;

-- 指定したDB内にあるテーブルの一覧を表示
SHOW TABLES;
  • SHOW DATABASES：現在MySQLに存在するDB（データベース）の一覧が表示される。
  • USE データベース名：これ以降の操作をどのDBに対して行うかを指定する。
  • SHOW TABLES：USE で選択したDB内のテーブル一覧が表示される。

4. 利用する DB を指定する方法

テーブルを作ったりデータを入れたりする前に、必ず USE でDBを選択します。選択しないまま操作しようとするとエラーになります（後述のエラーを参照）。

この研修では lesson という名前のDBを使用します。

-- DB が存在しない場合は先に作成する
CREATE DATABASE lesson;

-- 利用するDBを指定する
USE lesson;

-- 選択できたか確認（現在のDB名が表示される）
SELECT DATABASE();

ポイント：SQL を実行するたびに接続先のDBが記憶されているので、一度 USE すれば以降は同じセッション中ずっと有効です。

MySQL を再起動したり、別のコマンドプロンプトで接続し直したりすると、再度 USE が必要になります。

6. エラーの見方と対処方法

よく遭遇するメッセージを確認しておきましょう。

ERROR 1046 (3D000): No database selected

意味：操作対象のDBが選択されていない。

原因：MySQL に接続してすぐ USE を実行せずにテーブル操作を行おうとした。

対処：先に USE lesson; を実行してから、もう一度やり直す。

USE lesson;   -- これを先に実行してから操作を続ける
Query OK, 1 row affected

意味：正常にSQLが実行された。

内容：「1行が処理された（INSERT / UPDATE / DELETE など）」という成功メッセージ。数字は処理された行数を表す。

  • Query OK, 1 row affected：1件のINSERT / UPDATE / DELETE が成功した。
  • Query OK, 10 rows affected：10件まとめて処理が成功した（INSERT などのまとめ実行）。
  • Query OK, 0 rows affected：条件に一致する行がなかった（WHEREの値が存在しないなど）。エラーではないが注意が必要。
ERROR 1062 (23000): Duplicate entry ... for key 'PRIMARY'

意味：主キーが重複している。

原因：同じ item_id や emp_id を2回 INSERT しようとした。

対処：DROP TABLE IF EXISTS テーブル名; でテーブルごと削除してから、流し込みSQLを最初から実行し直す。
