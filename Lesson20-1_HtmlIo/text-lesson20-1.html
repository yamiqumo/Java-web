第20回：HTML基礎（I/O前半）〜フォームとテーブルで「入れる」「出す」をつかむ〜

1. 復習：これまでの「入出力」とこれからの「画面I/O」

これまでの自販機プログラムでは、次のような 入出力（I/O） を扱ってきました。

  • 入力：コンソールから Scanner で数字や文字列を読む
  • 処理：Java の変数やメソッドで在庫・金額を計算する
  • 出力：System.out.println でメッセージを表示する

カバ先生：「今までは コンソール → Java → コンソール という世界でした。これからは、ブラウザの画面 → サーバ側のJava → 画面に戻す という新しい『通り道』を作っていきます。」

ユウタ：「今まで println していた部分が、これからは画面のどこに出てくるのか、ってことですね。」

ナナコ：「今日はまず、サーバのことは置いておいて、『入力の箱』と『出力の箱』としてのフォームとテーブルだけをしっかり作る回なんですね。」

2. この回の学習目標

  • HTML だけで、商品登録フォーム＋ダミーの一覧テーブルを完成させる。
  • <form> / <input> / <button> / <table> / <tr> / <th> / <td> の役割を説明できる。
  • フォームの name 属性 が、将来 Servlet の getParameter につながる「ラベル」だとイメージできる。

3. 入力の箱：フォームの型

まずは、商品名と価格を入力するためのフォームを作ります。完成形は次のとおりです。

<form action="/dummy" method="post">
  <label>商品名:
    <input type="text" name="itemName">
  </label><br>

  <label>価格:
    <input type="number" name="price">
  </label><br>

  <button type="submit">登録</button>
</form>

ここでは、サーバ側はまだ作っていないので、action="/dummy" として「とりあえずどこかに送る」イメージだけ持っておきます。

  • <form> … 1枚の「申込書」全体。
  • action … 申込書を送る先（URL）。
  • method="post" … 送り方（POSTで送る）。
  • name="itemName" / name="price" … 将来、Servlet の getParameter("itemName") などで取り出すときの「ラベル」。

4. 出力の箱：テーブルの型

次に、登録された商品を一覧で表示するための テーブル を作ります。まずはダミーデータ 1件でOKです。

<table border="1">
  <tr>
    <th>ID</th>
    <th>商品名</th>
    <th>価格</th>
  </tr>
  <tr>
    <td>1</td>
    <td>サンプル商品</td>
    <td>100</td>
  </tr>
</table>
  • <tr> … 横1行（レコード1件やヘッダ1行）。
  • <th> … 見出しセル（列の名前）。
  • <td> … データセル（ID=1 の商品などの値）。

5. ハンズオン：フォーム＋テーブルを1画面に並べる

最後に、さきほどのフォームとテーブルを 1つの画面 に並べてみましょう。ここでは、レイアウトを単純にするために、フォーム → テーブルの縦並びにします。

<h2>商品登録フォーム</h2>

<form action="/dummy" method="post">
  <label>商品名:
    <input type="text" name="itemName">
  </label><br>

  <label>価格:
    <input type="number" name="price">
  </label><br>

  <button type="submit">登録</button>
</form>

<h2>商品一覧（サンプル）</h2>

<table border="1">
  <tr>
    <th>ID</th>
    <th>商品名</th>
    <th>価格</th>
  </tr>
  <tr>
    <td>1</td>
    <td>サンプル商品</td>
    <td>100</td>
  </tr>
</table>

6. まとめ

  • フォームは「申込書」、テーブルは「一覧表」。どちらも データの通り道の箱 である。
  • フォームの name 属性は、将来 Servlet が getParameter で値を取り出すときの「ラベル」になる。
  • テーブルの <tr> / <th> / <td> は、将来の「レコード」と「カラム」に対応する。
