第2回：JDBC接続の基本〜JavaからDBへ「橋」をかける〜

1. ① 概要：接続 → 準備 → DB へのアクセス → クローズ

今日は、Java から DB に命令を出すための入口（JDBC）を、短い手順として固める回です。

この回の全体の流れ（型）は次のとおりです。細部はまだ追わず、順番だけ頭に入れてください。

Connection（接続）→ PreparedStatement（SQL 準備・? へのバインド）
→ executeQuery / executeUpdate（DB へのアクセス＝実行）
→ ResultSet（SELECT のとき結果を読む）／更新件数を受け取る
→ try-with-resources または finally でリソースをクローズ

SELECT なら executeQuery、INSERT/UPDATE/DELETE なら executeUpdate と覚えておくと、後の図・コードと対応しやすくなります。

Java と SQL（DB）の位置づけ：SQL は「言語」、JDBC は「Java からその言語を運ぶための橋（API）」です。

2. ② 工程を追う（学習目標・用語・図）

2-1. 学習目標（手順を覚える）

この回では、JDBC の基本パイプラインを順番通りに言えること、と安全側の書き方（? とバインド）を掴むことを目指します。

JDBC の4ステップを順番で説明できる

接続（Connection）→ SQL 準備（PreparedStatement）→ 実行（execute*）→ 結果（ResultSet）の流れを、図とあわせて説明できる。

  • Connection：DB への接続を作る。
  • PreparedStatement：SQL を安全に組み立てる（? に値をバインド）。
  • executeQuery / executeUpdate：SELECT と更新系で使い分ける。
  • ResultSet：SELECT の結果を1行ずつ読む。
PreparedStatement を使う理由を一言で言える

値を文字列連結で SQL に埋め込まず、? に後から値を渡すことで、SQL インジェクション（悪意ある入力の混入）を避けやすくする、という説明ができる。

手順の型（写経テンプレ）

次の並びを、自分の言葉でも追えること。

Connection ▶ PreparedStatement ▶ executeQuery / executeUpdate ▶ ResultSet

2-2. 用語の確認（最小）

  • JDBC：Java から DB にアクセスするための標準 API（ドライバ経由で DBMS と会話する）。
  • Connection：DB への接続（セッション）を表すオブジェクト。
  • PreparedStatement：SQL を一度形にし、? に値を後から埋めるためのオブジェクト。
  • ResultSet：SELECT の結果（表のイメージ）を、カーソルを進めながら読むオブジェクト。
  • ドライバ（mysql-connector-j など）：JDBC の命令を、各 DBMS が理解できる形に変換する部品（クラスパスに追加する）。

2-3. 図でつかむ（4ステップ・安全性・読み方）

4ステップ全体像

JDBC は「接続 → 準備 → 実行 → 解放」をこの順番で回す、という全体像をまずイメージしてください。

DB 接続 URL の分解

URL の中で「どのDBに、どのユーザでつなぐか」を一つずつ分解して押さえましょう。

実行メソッドの分岐

3. ③ ソースコードの説明

3-1. 参考：try-catch-finally と手動 close（昔の書き方）

JDBC は Connection / PreparedStatement / ResultSet を必ず閉じる必要があります。昔は finally で close を書く形が定番でした。

ただし、この形はコードが長くなりやすいので、Java 7 以降は try-with-resources（次の節の形）を基本にします。この例では説明を簡単にするため、close は Connection（con）だけを示します。

import java.sql.*;

public class LegacyCloseSample {
	public static void main(String[] args) {
		Connection con = null;
		PreparedStatement ps = null;
		ResultSet rs = null;

		try {
			con = DriverManager.getConnection("jdbc:mysql://localhost:3306/sample_db?useSSL=false&serverTimezone=Asia/Tokyo", "root", "password");
			ps = con.prepareStatement("SELECT item_id FROM items WHERE category = ?");
			ps.setString(1, "drink");
			rs = ps.executeQuery();
			while (rs.next()) {
				System.out.println(rs.getInt("item_id"));
			}
		} catch (SQLException e) {
			e.printStackTrace();
		} finally {
			if (con != null) {
				try {
					con.close();
				} catch (SQLException e) {
					e.printStackTrace();
				}
			}
		}
	}
}

サンプル1：SELECT（executeQuery）

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

public class ItemsSelectSample {

	public static void main(String[] args) {
		String url = "jdbc:mysql://localhost:3306/sample_db?useSSL=false&serverTimezone=Asia/Tokyo";
		String user = "root";
		String password = "password";

		String sql = "SELECT item_id, category, item_name, unit_price FROM items WHERE category = ? ORDER BY item_id";

		try (
			Connection conn = DriverManager.getConnection(url, user, password);
			PreparedStatement ps = conn.prepareStatement(sql);
		) {
			ps.setString(1, "drink");

			try (ResultSet rs = ps.executeQuery()) {
				while (rs.next()) {
					int itemId = rs.getInt("item_id");
					String category = rs.getString("category");
					String itemName = rs.getString("item_name");
					int unitPrice = rs.getInt("unit_price");

					System.out.println(itemId + " | " + category + " | " + itemName + " | " + unitPrice);
				}
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
}

		}
	}

3-3. サンプルと対応付け（JDBC URL・Setter・Getter・カーソル）

上の サンプル1（SELECT） の各行が、② の図のどこに当たるかを対応づけます。

  • JDBC URL（String url）：jdbc:mysql://ホスト:ポート/DB名 に続く ? 以降は接続オプション（SSL やタイムゾーンなど）。user / password は getConnection の第2・第3引数で渡します。
  • Setter（バインド）：SQL 中の ? に、setString(番号, 値) や setInt で値を安全に流し込みます。番号は SQL 上の ? の左から 1, 2, … です。
  • Getter（結果の取り出し）：ResultSet に対して getInt("列名") / getString("列名") などで、現在の行のセルを Java の型で取り出します。
  • カーソル：ResultSet は「表」全体ではなく、今どの行を指しているかというカーソルを持ちます。初期位置はデータ行の手前で、next() が true のあいだ、1行ずつ進めながら getter で読みます。

6. まとめ（①の型を思い出す）

最後に、① で示した JDBC の流れを短く復習します。

1) Connection：DB に接続する
2) PreparedStatement：SQL を準備し、? に値をバインドする
3) 実行：SELECT は executeQuery／更新は executeUpdate
4) ResultSet：SELECT の結果を next と get* で読む
