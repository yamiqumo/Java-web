2. 学習目標（手順を覚える）

この回では、JDBC の基本パイプラインを順番通りに言えること、と安全側の書き方（? とバインド）を掴むことを目指します。

JDBC の4ステップを順番で説明できる

接続（Connection）→ SQL 準備（PreparedStatement）→ 実行（execute*）→ 結果（ResultSet）の流れを、図とあわせて説明できる。

  • Connection：DB への接続を作る。
  • PreparedStatement：SQL を安全に組み立てる（? に値をバインド）。
  • executeQuery / executeUpdate：SELECT と更新系で使い分ける。
  • ResultSet：SELECT の結果を1行ずつ読む。
PreparedStatement を使う理由を一言で言える

値を文字列連結で SQL に埋め込まず、? に後から値を渡すことで、SQL インジェクション（悪意ある入力の混入）を避けやすくする、という説明ができる。

手順の型（写経テンプレ）

次の並びを、自分の言葉でも追えること。

Connection ▶ PreparedStatement ▶ executeQuery / executeUpdate ▶ ResultSet

3. 写経パート（JDBC の型を体に入れる）

3-1. 用語の確認（最小）

  • JDBC：Java から DB にアクセスするための標準 API（ドライバ経由で DBMS と会話する）。
  • Connection：DB への接続（セッション）を表すオブジェクト。
  • PreparedStatement：SQL を一度形にし、? に値を後から埋めるためのオブジェクト。
  • ResultSet：SELECT の結果（表のイメージ）を、カーソルを進めながら読むオブジェクト。
  • ドライバ（mysql-connector-j など）：JDBC の命令を、各 DBMS が理解できる形に変換する部品（クラスパスに追加する）。

3-2b. 参考：try-catch-finally と手動 close（昔の書き方）

JDBC は Connection / PreparedStatement / ResultSet を必ず閉じる必要があります。昔は finally で close を書く形が定番でした。

ただし、この形はコードが長くなりやすいので、Java 7 以降は try-with-resources（次の節の形）を基本にします。

import java.sql.*;

public class LegacyCloseSample {
	public static void main(String[] args) {
		Connection conn = null;
		PreparedStatement ps = null;
		ResultSet rs = null;

		try {
			conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/sample_db?useSSL=false&serverTimezone=Asia/Tokyo", "root", "password");
			ps = conn.prepareStatement("SELECT item_id FROM items WHERE category = ?");
			ps.setString(1, "drink");
			rs = ps.executeQuery();
			while (rs.next()) {
				System.out.println(rs.getInt("item_id"));
			}
		} catch (SQLException e) {
			e.printStackTrace();
		} finally {
			if (rs != null) try { rs.close(); } catch (SQLException ignored) {}
			if (ps != null) try { ps.close(); } catch (SQLException ignored) {}
			if (conn != null) try { conn.close(); } catch (SQLException ignored) {}
		}
	}
}

3-3. Java サンプル（items：category を ? で検索）

次のサンプルは、接続情報（URL / ユーザー / パスワード）は自分の環境に合わせて置き換えてください。DB 側に items がある前提です（前回の DDL を流し込んでおくとスムーズです）。

依存ライブラリ（例：mysql-connector-j）はプロジェクトに追加し、実行時のクラスパスに含めます。

先に環境構築：JAR のDL（learningBOX）→ Eclipse の外部JAR追加 → 接続確認（SampleConnection）までを先に済ませてください。環境構築ページ

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

public class ItemsSelectSample {

	public static void main(String[] args) {
		String url = "jdbc:mysql://localhost:3306/sample_db?useSSL=false&serverTimezone=Asia/Tokyo";
		String user = "root";
		String password = "password";

		String sql = "SELECT item_id, category, item_name, unit_price FROM items WHERE category = ? ORDER BY item_id";

		try (
			Connection conn = DriverManager.getConnection(url, user, password);
			PreparedStatement ps = conn.prepareStatement(sql);
		) {
			ps.setString(1, "drink");

			try (ResultSet rs = ps.executeQuery()) {
				while (rs.next()) {
					int itemId = rs.getInt("item_id");
					String category = rs.getString("category");
					String itemName = rs.getString("item_name");
					int unitPrice = rs.getInt("unit_price");

					System.out.println(itemId + " | " + category + " | " + itemName + " | " + unitPrice);
				}
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
}

3-4. Web アプリとしてデプロイしたときの置き場所（Tomcat / webapps）

サーブレット/JSP として Tomcat にデプロイすると、「どのファイルがどこに置かれるか」が決まっています。図の対応を押さえると、404 やクラスが見つからない系のトラブルが減ります。

  • webapps/myapp/：コンテキストルート（URL の /myapp）。
  • index.jsp / css / js / images：ブラウザから直接アクセスされうる公開リソース。
  • WEB-INF/：外部 URL から直接アクセスできない領域（重要）。
  • WEB-INF/web.xml：デプロイメント記述子（Servlet 設定）。
  • WEB-INF/classes/：コンパイル済み .class（パッケージ階層のまま配置）。
  • WEB-INF/lib/：アプリ固有の .jar ライブラリ。

5. ラボ（10分チェックリスト）

10分で終わる最小確認（講師のOKが出るまで次に進めない、の運用にしてください）

  1. JDBC URL（ホスト名・ポート・DB名）が自分の環境と一致している。
  2. MySQL ドライバ（例：mysql-connector-j）がプロジェクトに入り、実行できる。
  3. SELECT が executeQuery で1回通り、コンソールに行が出る。
  4. ? のバインド（例：setString(1, "drink")）を1か所変えて、結果が変わることを確認する。
  5. 例外が出たら、スタックトレースの先頭メッセージを読み、接続／SQL／権限のどれかに切り分ける。
  6. 余力がある人だけ：INSERT を executeUpdate で1回通し、次の SELECT で増えたことを確認する。

6. まとめ（写経で締める）

最後に、JDBC の4ステップを短い写経で締めます。

1) Connection：DB に接続する
2) PreparedStatement：SQL を準備し、? に値をバインドする
3) 実行：SELECT は executeQuery／更新は executeUpdate
4) ResultSet：SELECT の結果を next と get* で読む
