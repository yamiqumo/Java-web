問題1：チェック例外と非チェック例外（RuntimeException 系）について、コンパイラが呼び出し側に「処理の記述」を強く求めるかどうかの観点で、それぞれ一言ずつ説明してください。

ヒント

「メソッドに throws が付く」「呼び出し側が try-catch か throws を書く必要がある」がどちらに当てはまるかで整理します。

解答例

【解説・確認用】

チェック例外：呼び出し側に try-catch または throws など、例外の処理経路をコンパイラがチェックしやすい（「処理を書かせる」側に寄せる）。

非チェック例外（RuntimeException 系）：原則としてコンパイラは呼び出し側に try-catch を強制しない（実行時の前提崩れ・バグ寄りとして扱う）。

問題2：次のコード片はコンパイルエラーになります。理由を「catch の順序」に関連づけて説明してください。

try {
	work();
} catch (Exception e) {
	System.out.println("A");
} catch (IllegalArgumentException e) {
	System.out.println("B");
}
ヒント

IllegalArgumentException は Exception のサブクラスです。上からマッチするとき、2つ目の catch は到達しますか。

解答例

【解説・確認用】

catch は上から順に「捕捉できる型か」を見る。Exception は IllegalArgumentException より広い親なので、先に Exception を書くと
すべての例外がそこで捕捉される扱いになり、後ろの IllegalArgumentException には到達しない（到達不能コードとしてエラー）。

マウスで全選択して Eclipse やメモ帳へドラッグしてください：

問題3：次の main の実行で、コンソールに表示される行を順番どおりすべて書いてください。

public class FinallyOrderSample {

	public static void main(String[] args) {
		try {
			System.out.println("try");
			throw new IllegalArgumentException("demo");
		} catch (IllegalArgumentException e) {
			System.out.println("catch");
		} finally {
			System.out.println("finally");
		}
		System.out.println("after");
	}
}
ヒント

例外が出て catch に入ったあとも、finally は通ります。そのあと try ブロック全体の外の行へ進みます。

解答例

【解説・確認用】

try
catch
finally
after

問題5：社員の年齢をセットする Employee です。空欄（a）（b）に入る語句を答え、（c）に適切な型の例外オブジェクトを生成する式を1つ書いてください（非チェックでよい）。

public class Employee {

	private int age;

	public Employee() {}

	public void setAge(int age) {
		if (age < 0) {
			(a) new (b)("年齢は0以上");
		}
		this.age = age;
	}

	public int getAge() {
		return age;
	}
}
ヒント

不正な引数に対して投げる定番の非チェック例外クラス名を思い出してください。(a) はキーワード1語です。

解答例

【解説・確認用】

(a) throw
(b) IllegalArgumentException
(c) new IllegalArgumentException("年齢は0以上")

完成形のメソッド例：
	public void setAge(int age) {
		if (age < 0) {
			throw new IllegalArgumentException("年齢は0以上");
		}
		this.age = age;
	}
